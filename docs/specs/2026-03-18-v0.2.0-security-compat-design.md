# X2Mail v0.2.0 — Security Hardening & NC 33/PHP 8.4 Compatibility

**Date:** 2026-03-18
**Status:** Completed — released as v0.2.0
**Scope:** Security fixes, PHP 8.4 deprecation patches, deprecated NC API removal

---

## Overview

Audit-driven release addressing 3 HIGH security issues, PHP 8.4 deprecations in SM Core v2.38.2, and 28 deprecated \OC::$server calls. No new features.

---

## 1. Security Fixes (HIGH)

### 1.1 XSS in iframe src — templates/index.php

**Problem:** $_GET['target'] flows unescaped into iframe src attribute. Crafted URL like ?target=" onload="alert(1)" injects arbitrary HTML.

**Fix:** Escape output with htmlspecialchars(). Also sanitize target param in PageController with strict regex ([a-zA-Z0-9/\-_] only).

### 1.2 Arbitrary file require via custom_config_file — InstallStep.php

**Problem:** require $customConfigFile where path comes from oc_appconfig. Weak validation (only checks for :). Allows arbitrary code execution with any readable PHP file.

**Fix:** Restrict to appdata_x2mail/ directory with realpath() validation.

### 1.3 CSRF disabled on POST endpoints — PageController.php

**Problem:** appPost() and indexPost() have #[NoCSRFRequired] on POST handlers. These proxy all SnappyMail API calls.

**Decision:** Accepted trade-off — SnappyMail internal AJAX does not carry NC CSRF tokens. SM has its own CSRF layer internally. Add code comment documenting this decision. Not fixable without modifying SM core JS.

---

## 2. Security Fixes (MEDIUM)

### 2.1 $_POST/$_GET/$_SERVER direct access — FetchController, PageController

**Problem:** setAdmin() and setPersonal() read $_POST directly (15+ occurrences). index() reads $_SERVER['QUERY_STRING'] and $_GET['target'].

**Fix:** Replace all with $this->request->getParam() / framework request methods.

### 2.2 Unvalidated app_path — FetchController.php

**Problem:** Admin can set app_path to arbitrary path in SM config.

**Fix:** Validate starts with / and contains no ://.

### 2.3 SSRF via preflight check — SetupController.php

**Decision:** Accepted risk — admin-only endpoint. Document in code. IMAP/SMTP connectivity checks require connecting to user-specified hosts by design.

### 2.4 Admin password displayed in plain text — AdminSettings.php

**Fix:** Show password only if admin_password.txt exists, masked with reveal button or delete file after first display.

### 2.5 CSP header_remove() — SnappyMailHelper.php

**Decision:** Accepted — SM handles its own CSP internally. Add comment.

### 2.6 CSP allows script evaluation — ContentSecurityPolicy.php

**Decision:** Required by SM JavaScript. Add comment documenting reason.

### 2.7 Weak password salt (md5 of email) — SnappyMailHelper.php

**Decision:** SM architectural issue. X2Mail is OIDC-first, password path is fallback only. Document.

### 2.8 Timing-unsafe admin key comparison — PageController.php

**Fix:** Loose comparison to hash_equals() for admin panel key.

### 2.9 Missing port validation in saveSetup — SetupController.php

**Fix:** Add 1-65535 range check (same as preflightCheck already has).

## 4. Deprecated NC API Removal (28 occurrences)

Replace \OC::$server->getXxx() with \OCP\Server::get(Interface::class) or constructor DI.

### 4.1 SnappyMailHelper.php (15 occurrences + 1 OC_User)

Static utility class — replace with \OCP\Server::get() throughout:
- getSystemConfig()->getValue() to Server::get(IConfig::class)->getSystemValue()
- getUserSession() to Server::get(IUserSession::class)
- getSession() to Server::get(ISession::class)
- getConfig() to Server::get(IConfig::class)
- getAppManager() to Server::get(IAppManager::class)
- OC_User::isAdminUser() to Server::get(IGroupManager::class)->isAdmin()

### 4.2 Application.php (4 occurrences)

boot() method — \OC::$server->getSession() in event listeners:
- Get ISession from $context->getServerContainer()->get() and capture in closures

### 4.3 InstallStep.php (4 occurrences)

IRepairStep has no constructor DI — use \OCP\Server::get():
- All \OC::$server->get/getAppManager to \OCP\Server::get()

### 4.4 AdminSettings.php (3 occurrences + 1 OC_User)

Add IUserSession, IURLGenerator, IAppManager, IGroupManager to constructor.

### 4.5 PageController.php (1 occurrence)

Add IConfig to constructor.

### 4.6 FetchController.php (1 occurrence)

Add ?string $userId constructor param (auto-injected by NC DI).

### 4.7 Command/Status.php + Setup.php (3 occurrences)

Add IAppManager to constructors.

### 4.8 Remaining files (3 occurrences)

- ContentSecurityPolicy.php: use \OCP\Server::get() (no public API for nonce manager)
- RainLoop.php: getSystemConfig() to \OCP\Server::get(IConfig::class)->getSystemValue()
- LogService.php: getSystemConfig() to \OCP\Server::get(IConfig::class)->getSystemValue()
